=============================================
DESCRIPCIÓN DEL SISTEMA - SALÓN SOCIAL
Sistema de Gestión Residencial
=============================================

1. ¿QUÉ HACE EL SISTEMA?
--------------------------
El sistema permite gestionar el salón social de una unidad residencial
llamada "Vivimos Todos". Administra usuarios (titulares de apartamentos),
el inventario de insumos del salón, y prepara la estructura para un
futuro módulo de reservas.

2. FLUJO COMPLETO DEL SISTEMA
-------------------------------
a) LOGIN:
   - El usuario ingresa su número de apartamento y contraseña
   - El sistema valida las credenciales contra la base de datos MySQL
   - Las contraseñas están encriptadas con bcrypt
   - Si es válido, se genera un token JWT con duración de 8 horas
   - El token se almacena en localStorage del navegador

b) PANEL PRINCIPAL (Dashboard):
   - Muestra estadísticas generales: total usuarios, activos, inactivos, insumos
   - Resumen del estado de los módulos del sistema
   - Acceso rápido a las funcionalidades

c) GESTIÓN DE USUARIOS (solo Administrador):
   - Crear nuevo titular de apartamento
   - Editar datos del titular (cambio de titular)
   - Activar/desactivar usuarios (borrado lógico)
   - Asignar roles: administrador, residente, supervisor

d) GESTIÓN DE INVENTARIO:
   - CRUD completo de insumos del salón
   - Validaciones: no negativos, disponible ≤ total
   - Barra visual de disponibilidad con colores semáforo
   - Administradores y supervisores pueden editar
   - Solo administradores pueden eliminar

e) RESERVAS (Preparado para futuro):
   - Interfaz visual con calendario del mes actual
   - Lista de funcionalidades próximas
   - Estructura preparada para integración

3. ARQUITECTURA MVC
--------------------
BACKEND (Node.js + Express):
  /config        → Configuración de base de datos (pool MySQL)
  /models        → Acceso a datos y consultas SQL
  /controllers   → Lógica del negocio
  /routes        → Definición de endpoints API REST
  /middlewares   → Autenticación JWT y manejo de errores

FRONTEND (React + Vite):
  /src/context   → Estado global (autenticación)
  /src/services  → Consumo de API con Axios
  /src/pages     → Páginas principales
  /src/components → Componentes reutilizables (modales, layout)

4. RELACIÓN ENTRE MÓDULOS
---------------------------
- El módulo de autenticación protege todas las rutas del backend
- El contexto de autenticación en el frontend gestiona el estado del usuario
- Los roles determinan qué funcionalidades puede ver cada usuario
- El servicio API centraliza todas las peticiones HTTP con interceptores

5. DECISIONES TÉCNICAS
-----------------------
- MySQL con pool de conexiones para eficiencia
- bcrypt para encriptación de contraseñas
- JWT para autenticación stateless
- Axios con interceptores para manejo automático de tokens
- Bootstrap para diseño responsive obligatorio
- Borrado lógico (campo estado) en lugar de eliminación física
- id_apartamento como clave primaria (no autoincremental)

6. FUNCIONALIDADES FUTURAS
----------------------------
- Módulo completo de reservas con calendario interactivo
- Sistema de notificaciones por correo
- Reportes y estadísticas avanzadas
- Historial de cambios de titulares
- Gestión de pagos por uso del salón
